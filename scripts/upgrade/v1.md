# Upgrading to the latest suede layout

This migrates a repository created with an earlier version of the suede workflow
onto the current subrepo layout, where `.suede/core` and `.github/workflows` are
each vendored — via [git-subrepo](https://github.com/ingydotnet/git-subrepo) —
from a dedicated branch of the suede library.

Two branches get rewired:

| Branch    | `.github/workflows` from         | `.suede/core` from          |
| --------- | -------------------------------- | --------------------------- |
| `release` | `dependency/release/workflows`   | `dependency/release/core`   |
| `main`    | `dependency/main/workflows`      | `dependency/main/core`      |

Along the way some now-obsolete generated files are dropped: the old per-branch
workflow on each side, plus the one-shot `initialize.yml` on `main`.

> This used to be a single script. It was fiddly enough that the steps are now
> spelled out manually — run them yourself and watch each one. Do the **release**
> branch first, then **main** (main pulls from the rebuilt release).

## Before you start

- Have `git-subrepo` installed (`git subrepo --version`).
- Start from a clean working tree on an up-to-date `main`:

```bash
git checkout main && git pull origin main
```

> [!NOTE]
> Each rebuild **replaces** the whole `.github/workflows` folder. If you've added
> your own workflow files there, copy them somewhere safe first and add them back
> after the clone — the steps below do not preserve them automatically.

---

## 1. Rebuild the `release` branch

### 1a. Check out `release` and update it

```bash
git checkout release && git pull
```

### 1b. Remove the old workflows folder

It's regenerated from the subrepo, so drop it (this also removes the obsolete
`subrepo-pull-into-main.yml`). `git subrepo clone` requires a clean tree, so
commit the removal.

```bash
git rm -r .github/workflows && git commit -m "suede upgrade: remove old workflows"
```

### 1c. Vendor the current release workflows

```bash
git subrepo clone https://github.com/pmalacho-mit/suede.git .github/workflows --branch=dependency/release/workflows
```

### 1d. Vendor the current release `.suede/core`

```bash
git subrepo clone https://github.com/pmalacho-mit/suede.git .suede/core --branch=dependency/release/core
```

### 1e. Push `release`

```bash
git push origin release
```

### All of step 1 as one command

```bash
git checkout release && git pull \
  && git rm -r .github/workflows && git commit -m "suede upgrade: remove old workflows" \
  && git subrepo clone https://github.com/pmalacho-mit/suede.git .github/workflows --branch=dependency/release/workflows \
  && git subrepo clone https://github.com/pmalacho-mit/suede.git .suede/core --branch=dependency/release/core \
  && git push origin release
```

---

## 2. Rebuild the `main` branch

### 2a. Check out `main`

```bash
git checkout main
```

### 2b. Sync the `release/` folder to the rebuilt release branch

```bash
git subrepo pull release
```

### 2c. Remove the old workflows folder

Drops the obsolete `subrepo-push-release.yml` along with it; commit so the tree
is clean for the clone.

```bash
git rm -r .github/workflows && git commit -m "suede upgrade: remove old workflows"
```

### 2d. Vendor the current main workflows

```bash
git subrepo clone https://github.com/pmalacho-mit/suede.git .github/workflows --branch=dependency/main/workflows
```

### 2e. Drop the one-shot `initialize.yml`

The main workflows subrepo ships `initialize.yml`, which is only needed once at
repo creation — remove it.

```bash
git rm .github/workflows/initialize.yml && git commit -m "suede upgrade: drop initialize.yml"
```

### 2f. Vendor the current main `.suede/core`

```bash
git subrepo clone https://github.com/pmalacho-mit/suede.git .suede/core --branch=dependency/main/core
```

### 2g. Repoint the README install links

The release install script moved from `scripts/install-release.sh` to the nested
`scripts/install/release.sh`. Skip this step if your `README.md` has no
`install-release` references.

```bash
sed -i -e 's#suede\.sh/install-release#suede.sh/install/release#g' \
       -e 's#scripts/install-release\.sh#scripts/install/release.sh#g' README.md \
  && git commit -am "suede upgrade: point README at install/release"
```

### 2h. Push `main`

```bash
git push origin main
```

### All of step 2 as one command

> Drop the `sed ... && git commit` line if your `README.md` has no
> `install-release` links (otherwise the empty commit aborts the chain).

```bash
git checkout main \
  && git subrepo pull release \
  && git rm -r .github/workflows && git commit -m "suede upgrade: remove old workflows" \
  && git subrepo clone https://github.com/pmalacho-mit/suede.git .github/workflows --branch=dependency/main/workflows \
  && git rm .github/workflows/initialize.yml && git commit -m "suede upgrade: drop initialize.yml" \
  && git subrepo clone https://github.com/pmalacho-mit/suede.git .suede/core --branch=dependency/main/core \
  && sed -i -e 's#suede\.sh/install-release#suede.sh/install/release#g' \
            -e 's#scripts/install-release\.sh#scripts/install/release.sh#g' README.md \
  && git commit -am "suede upgrade: point README at install/release" \
  && git push origin main
```

---

Once both branches are pushed, `release` and `main` each vendor `.suede/core`
and `.github/workflows` from the current suede library branches.
